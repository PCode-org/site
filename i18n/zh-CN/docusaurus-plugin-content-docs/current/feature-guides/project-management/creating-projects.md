# 创建项目

创建项目是使用 PCode 的第一步，本指南将帮助您创建和管理项目。

## 创建方式

PCode 支持多种创建项目的方式：

### 从现有目录创建

将现有的代码目录创建为 PCode 项目：

1. 点击 **创建项目** 按钮
2. 选择 **从现有目录创建**
3. 浏览并选择目录路径
4. 输入项目名称
5. 点击 **创建**

### 克隆远程仓库

从 Git 仓库克隆创建项目：

1. 点击 **克隆项目** 按钮
2. 输入远程仓库 URL
3. 选择本地保存路径
4. 输入项目名称（可选）
5. 点击 **克隆**

### 创建新项目

创建一个新的空白项目：

1. 点击 **创建项目** 按钮
2. 选择 **创建新项目**
3. 输入项目名称
4. 选择项目路径
5. 点击 **创建**

## 项目信息

### 项目名称

- **必填字段**：项目必须有名称
- **命名规则**：
  - 支持中英文
  - 不支持特殊字符
  - 建议使用简短、描述性的名称
- **唯一性**：项目名称在您的账户中应唯一

### 项目描述

- **可选字段**：项目描述是可选的
- **建议内容**：
  - 项目的用途
  - 项目的技术栈
  - 项目的目标
- **长度限制**：最多 500 个字符

### 项目路径

- **本地项目**：选择本地文件系统路径
- **远程项目**：选择克隆后的本地路径
- **路径要求**：
  - 路径必须存在
  - 必须有读写权限
  - 建议选择空目录或项目根目录

## 项目类型

### 本地项目

直接访问本地文件系统的项目：

- **优点**：
  - 快速访问
  - 无需网络连接
  - 完整的文件操作权限

- **适用场景**：
  - 本地开发
  - 个人项目
  - 快速原型开发

### 远程项目

连接到 Git 远程仓库的项目：

- **优点**：
  - 支持版本控制
  - 团队协作
  - 代码备份和同步

- **适用场景**：
  - 团队项目
  - 开源项目
  - 需要版本控制的项目

## 项目配置

### Git 配置

对于 Git 项目，可以配置：

- **仓库 URL**：远程仓库的地址
- **分支**：默认分支名称
- **认证方式**：SSH 或 HTTPS
- **拉取策略**：自动拉取或手动拉取

### 项目范围

配置项目的可见性：

- **全局范围**：在全局视图中可见
- **项目范围**：仅在项目视图中可见
- **私有**：仅创建者可见

### 通知设置

配置项目相关的通知：

- **会话通知**：新会话通知
- **更新通知**：项目更新通知
- **错误通知**：错误和警告通知

## 创建后操作

项目创建后，建议进行以下操作：

### 初始化会话

创建第一个会话：

1. 点击 **创建会话** 按钮
2. 选择会话类型（聊天或提案）
3. 开始使用 PCode

### 配置设置

配置项目设置：

1. 打开项目详情
2. 配置项目范围
3. 设置通知偏好
4. 配置其他项目设置

### 邀请成员（可选）

如果是团队项目：

1. 打开项目设置
2. 选择 **成员管理**
3. 邀请成员加入
4. 分配成员角色

## 最佳实践

### 项目命名

- 使用描述性的名称
- 包含项目的技术栈或用途
- 避免使用特殊字符
- 保持名称简洁

### 项目组织

- 按用途分组项目
- 使用项目前缀或后缀分类
- 定期清理不使用的项目
- 归档已完成的项目

### 路径选择

- 选择易于访问的路径
- 避免使用系统路径
- 确保有足够的磁盘空间
- 考虑备份策略

## 故障排除

### 创建失败

如果项目创建失败：

1. **检查路径**：确保路径存在且有权限
2. **检查名称**：确保项目名称符合要求
3. **检查网络**：如果是远程项目，检查网络连接
4. **查看日志**：查看详细的错误日志

### 克隆失败

如果克隆远程仓库失败：

1. **验证 URL**：确保仓库 URL 正确
2. **检查权限**：确保有访问仓库的权限
3. **检查认证**：确保配置了正确的认证方式
4. **检查网络**：确保可以访问远程仓库

### 路径问题

如果遇到路径相关的问题：

1. **使用绝对路径**：避免使用相对路径
2. **检查权限**：确保有读写权限
3. **避免特殊字符**：路径中不要包含特殊字符
4. **检查磁盘空间**：确保有足够的磁盘空间
