# 变更：为项目创建文档添加截图

## 状态
ExecutionCompleted

## 为什么

当前「创建您的第一个项目」文档仅包含文字说明,缺乏可视化截图指引。用户难以直观理解 PCode 界面的操作流程和界面布局,增加了学习曲线和操作困惑。通过集成已有的截图资源,可以显著提升文档的易用性和用户体验。

## 变更内容

- **在 `docs/quick-start/create-first-project.md` 中添加截图引用**
  - 步骤 2: 在"访问项目页面"部分添加「点击新建项目按钮」截图
  - 步骤 2: 在"配置项目设置"部分添加「填写项目的一般信息」截图
  - 步骤 2: 在"保存项目"部分添加「项目列表中查看新建项目」截图
  - 步骤 3: 在"导航到项目详情"部分添加「点击项目 SDD tab」截图
  - 步骤 3: 在"运行 OpenSpec 初始化"部分添加「SDD 初始化完成后的状态」截图
  - 步骤 4: 在"编辑 project.md"部分添加「点击优化按钮优化 Project.md」截图

- **优化文档结构与截图布局**
  - 确保每个关键操作步骤后紧跟对应截图
  - 为截图添加清晰的 alt 描述文字
  - 使用 Docusaurus 标准图片语法: `![描述](/img/create-project/文件名.png)`
  - 保持截图顺序与操作流程一致

## UI 设计变更

不适用 - 本变更仅涉及文档内容增强,不涉及 UI 界面修改。

## 代码流程变更

不适用 - 本变更仅涉及文档内容更新,不涉及代码逻辑修改。

## 影响

### 影响的规范
- `docusaurus-site` - 修改「Quick Start Project Creation Documentation」需求以包含可视化截图要求

### 影响的代码
- `docs/quick-start/create-first-project.md` - 在相应章节添加 6 张截图引用

### 预期收益
- **用户体验提升**: 通过可视化截图降低用户学习曲线,使项目创建流程更易于理解和操作
- **文档质量增强**: 图文结合提升文档的专业性和可读性
- **减少支持成本**: 清晰的步骤指引减少用户在项目创建过程中的疑问和困惑

### 验证标准
- 本地开发服务器运行正常(`npm start`)
- 生产构建成功无错误(`npm run build`)
- 文档页面中所有 6 张截图正确显示
- 截图与文字描述逻辑对应准确
- 遵循 Docusaurus 的 Markdown 图片语法规范
- 图片路径使用绝对路径(`/img/create-project/`)
